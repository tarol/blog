前言

众所周知的是，函数是 ES 的一等公民，尤其是 ES6 发布后，随着箭头函数和类的加入，函数的 feature 日趋复杂。而现在，在 TC39 的 proposals 中，越来越多的 feature 是基于函数的，比如 decorators、 partial-application、 Pipeline Operator 等

我们从头到尾缕一遍函数的特性，包括最早的闭包，ES5 的 call、apply、bind，ES6 的箭头函数和类

例 2 的答案是 tarol，原因很简单，就是闭包的核心概念：当一个函数运行时，它可以访问定义函数时所在的作用域

为什么可以访问呢？因为函数在定义时，把当前的作用域作为内部属性（不开放给外部访问）固化在了这个函数的实例中，当函数运行的时候，把这个内部属性拿过来，用这个作用域作为父作用域，生成一个新的作用域链
